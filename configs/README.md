# configs/

Per-model openharn configs, generated by `tests/tune_model.py` (cross-platform)
or `tests/tune_model.sh` (Unix shell).

Each file is a plain `KEY=value` list — one var per line. `#` comments and
blank lines are ignored; lines without `=` are skipped. It is exactly the
winning `OPENHARN_*` flag combo for that model, plus a header comment with the
matching `llama-server` command.

Load it:

```sh
./target/debug/openharn . --config configs/<model>.conf
```

or let openharn auto-load it when `configs/<OPENHARN_MODEL>.conf` exists.

Example (`configs/LFM2.5-8B-A1B-APEX-I-Compact.conf`):

```
# openharn config for LFM2.5-8B-A1B-APEX-I-Compact
# generated by tests/tune_model.sh — best-scoring config (score 6/6, 41s)
# server: llama-server -m LFM2.5-8B-A1B-APEX-I-Compact.gguf --jinja --ctx-size 16384 -ngl 0 --host 127.0.0.1 --port 8080 --no-warmup
# run:    OPENHARN_MODEL=LFM2.5-8B-A1B-APEX-I-Compact ./target/debug/openharn . --config configs/LFM2.5-8B-A1B-APEX-I-Compact.conf
OPENHARN_PROMPT_TOOLS=1
OPENHARN_STRICT_TOOLS=1
```
